# NIVEAU AVANCÉ - Architectures complexes
# ============================================================================

## Exemple 10: E-commerce platform
Infrastructure e-commerce: 
- 5 instances EC2 t2.large 
- RDS MySQL db.m5.xlarge avec 500GB 
- S3 pour produits images 
- CloudFront CDN 
- Application Load Balancer 
- Security groups ports 80 et 443

## Exemple 11: Microservices sur containers
Architecture microservices:
- EKS cluster avec 15 pods
- RDS PostgreSQL multi-AZ db.r5.large
- DynamoDB pour cache
- S3 pour logs et backup
- API Gateway
- VPC 10.0.0.0/16 avec 3 subnets

## Exemple 12: Data analytics platform
Plateforme analytics:
- 3 EC2 t3.2xlarge pour processing
- RDS PostgreSQL db.r5.xlarge 1TB
- S3 data lake 10TB
- ElastiCache Redis pour cache
- VPC privé avec NAT Gateway

## Exemple 13: Mobile backend API
Backend mobile:
- API Gateway REST
- Lambda functions
- DynamoDB tables
- S3 pour user uploads
- CloudFront pour CDN
- Cognito pour authentication

## Exemple 14: Three-tier application
Application 3-tiers:
- Internet-facing ALB
- 4 EC2 t3.large web servers
- Internal NLB
- 3 EC2 t3.medium app servers
- RDS MySQL Multi-AZ db.m5.xlarge
- S3 pour static assets
- VPC avec subnets publics et privés
- Security groups pour chaque tier

## Exemple 19: Gaming backend

Infrastructure sécurisée:
- EC2 dans subnet privé

- S3 avec encryption et versioning

- VPC avec NAT Gateway
- CloudWatch logs enabled

## Exemple 23: Auto-scaling
Infrastructure élastique:

- CloudWatch alarms

## Exemple 24: Compliance requirements
Infrastructure conforme:
- EC2 encrypted volumes
- RDS Multi-AZ avec backups

- VPC flow logs
- KMS encryption keys

## Exemple 25: Performance optimisée
Infrastructure haute performance:

- Placement groups

## Exemple 26
J'aimerais déployer une application web moderne avec une architecture scalable. 
Il me faut plusieurs serveurs pour gérer la charge, une base de données relationnelle 
performante, du stockage pour les médias des utilisateurs, et bien sûr un load balancer 
pour distribuer le trafic. La sécurité est importante donc j'ai besoin de groupes de 
sécurité appropriés.

## Exemple 27
Pour notre startup, on a besoin d'une infrastructure cloud pas trop chère pour commencer. 
Un petit serveur devrait suffire, avec une base MySQL et un endroit pour stocker 
les fichiers des clients. On verra pour scaler plus tard.

## Exemple 28
Infrastructure enterprise: nous voulons une architecture trois tiers classique avec 
des serveurs web publics, des serveurs applicatifs privés, et une base de données 
hautement disponible. Le tout dans un VPC isolé avec des sous-réseaux publics et privés. 
Ajoutez aussi un CDN pour la distribution mondiale de notre contenu statique.
